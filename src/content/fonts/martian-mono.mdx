A monospace is a constraint dressed up as a font: every glyph, fat or thin, must occupy exactly the same width so that columns of code and data line up into a clean grid. **Martian Mono**, drawn by **Roman Shchedov** at the engineering studio **Evil Martians**, accepts that constraint and then quietly modernises it. It is the monospaced sibling of the studio’s Martian Grotesk, and it carries the same boxy, geometric, faintly extraterrestrial look — letters squared to the cell, curves tightened, descenders cropped.

What sets it apart from the usual coding faces is that it is variable, and on two axes at once. The weight runs the full distance from a hairline Thin to a heavy Extra Bold, while a width axis lets you compress or expand the design — all without ever breaking the fixed advance that makes a monospace a monospace. The result is a single file that can be a delicate caption, a dense terminal, or a wide, confident headline, while still holding every character in its column.

The detailing is built for the places monospace actually gets used. The zero is slashed so it can never be mistaken for a capital O, the figures are tabular by nature, and the whole alphabet keeps a generous x-height and open apertures so it stays sharp at the small sizes of an editor or a log. It reads as technical without tipping into the retro, typewriter nostalgia that many coding fonts lean on.

Released free and open source by a team that ships software for a living, Martian Mono is a monospace made by its own users — equally at home setting a snippet of code, a data dashboard, or the cool, machined captions of a product built by engineers.
